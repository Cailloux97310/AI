Algorithme Excelsior XW72
Détail technique de l'encodage bijectif v7 et du processus d'anonymisation.
Flux complet
ETAPE 1 — Anonymisation Pour chaque règle (orig, repl), triées par longueur desc : remplacer orig dans le texte (insensible à la casse) ETAPE 2 — Encodage XW72 rotation = 0, counter = 0 Pour chaque ligne : Pour chaque mot (séparé par espace) : Pour chaque caractère c (ASCII 32-126) : idx = (ord(c) - 32 + rotation) % 95 émettre TABLE[idx] counter++ ; si counter % 50 == 0 : rotation++, émettre [Rn] émettre WORDSEP émettre NEWLIN émettre [CRC:XXXX] (CRC16-CCITT sur la ligne ASCII)

Anonymisation — détail
Ordre d'application : les règles sont triées par longueur décroissante avant application. Cela évite qu'une règle courte ("Jean") ne masque le début d'une règle longue ("Jean Dupont").

Insensibilité à la casse : "acme", "ACME", "Acme" sont tous remplacés. Le remplacement est inséré tel quel (sans adaptation de casse).

Persistance : les règles peuvent être exportées en JSON et réimportées à la session suivante.
